  • gitflow, github flow, etc ..
  • fork vs same repo branching
  • document it! CONTRIBUTING.md

Command Line

  • custumize your shell
  • do not use the GUI
  • use aliases

Commit Often

  • commit each fix as a separate change
  • short commit message in the first line (50-72 character)
  • imperative mood in the subject line
  • reference the issue on the body of your commit message

Don't Pollute the Log

  • don't push to master any 'merge messages'
  • update your local development branch with git pull --rebase origin master
  • always Rebase over merge

Continuous Integration Practices

  • don’t check in broken code.
  • check in frequently
  • whenever you reach a point where the code is complete enough and doesn't break other things, it goes in.
  • master branch should always be deployable
  • over communicate: communication plays a primary role in continuous integration
  • don’t check in untested code
